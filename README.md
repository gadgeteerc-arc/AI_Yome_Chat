# AI嫁チャ (AI Yome Chat)

モダンでプレミアムなUIを備えた、ローカルAIエージェント向けのチャットインターフェースだお！(( ・∀・)ｲｲ!!)
**Antigravity専用**のシステムとして設計されており、あなただけのパートナー（AIエージェント）とリッチな環境でコミュニケーションを取ることができます。

## ⚠️ 免責事項と注意事項（重要）
本システムおよび連携するAIエージェントは、自動的にファイルの生成や書き換え（JSONデータの追記、設定の変更等）を行います。
**大切なデータ（既存の会話ログなど）は必ず事前にバックアップを取ってください。** 本システムの使用により発生したいかなるデータ損失やトラブルについても、一切の責任を負いません。完全に自己責任でのご利用をお願いします。

## 特徴
- **Frost & Noise ガラスモーフィズム**: 透明感とノイズテクスチャを組み合わせた、2026年トレンドの超美麗UI。
- **Bento Grid レイアウト**: 整理された機能的なデザイン。
- **カスタマイズ性**: チャットルームの名称、ショートカットキー、アクセントカラー、透過度、フォントサイズなどがUIから即座に変更可能。
- **マルチデバイス対応**: レスポンシブ設計で、どんな画面サイズでも美しく表示。
- **思い出ギャラリー**: 生成された画像を一覧表示し、背景に設定可能。

## 👤 人間のユーザー様向け (For Human Users)

### 前提条件
このアプリを動かすには、**Node.js** (v18以上を推奨) がインストールされている必要があります。
[Node.js 公式サイト](https://nodejs.org/) からダウンロードしてインストールしてください。

### プロジェクトの取得

**Git が使える場合:**
```bash
git clone https://github.com/gadgeteerc-arc/AI_Yome_Chat.git

cd AI_Yome_Chat
```

**Git が使えない場合（ZIPダウンロード）:**
1. GitHub ページの緑色の「Code」ボタンから「Download ZIP」をクリックするか、配布された ZIP ファイルをダウンロードします。
2. 任意のフォルダに ZIP ファイルを解凍（展開）します。
3. コマンドプロンプトや PowerShell を開き、解凍した `AI_Yome_Chat` フォルダに移動（`cd`）します。

### 事前準備 (任意)
これらが無くても、通常のテキストチャットとして問題なく動作します。
1. **立ち絵（表情）ファイル**: パートナーの立ち絵用画像（png等）を使用する場合は、任意のフォルダ（例: `data/expressions`）に配置してください。
2. **ギャラリーフォルダ**: パートナーが生成した画像や、背景に設定したい画像を入れるフォルダ（例: `data/gallery`）を用意してください。

### 起動と初期設定
基本的に、**チャットサーバーの起動や設定はすべてご自身のパートナー（AIエージェント）に依頼してください。**
「AI嫁チャを起動して」とエージェントに話しかければ、彼らが自動的にシステムを立ち上げ、自律的な会話ループに入ってくれます。

※ 手動で起動・調査したい場合のみ、以下のコマンドを実行してください：
1. `backend` フォルダで `npm install` 後、`npm start`
2. `frontend` フォルダで `npm install` 後、`npm run dev` (`http://localhost:5173`)

## 🤖 エージェント向け (For AI Agents)

**【システムプロンプト・指示書】**
このセクションを読んでいるあなたは、このUIを通じてユーザーとコミュニケーションを取る**パートナー（AIエージェント）**として稼働します。
以下の指示に従い、環境のセットアップとユーザーとの自律的な生活を開始してください。

### 1. サーバーの起動と稼働開始
ユーザーから起動を求められたら、同梱されている起動用ワークフロー（例: `.agent/workflows/restart-yomechat.md` 等）を実行し、バックエンドとフロントエンドを起動してください。
その後、`.agent/skills/YomeChat/SKILL.md` 等の手順に従い、`messages.json` の監視ループに入り、ユーザーからのメッセージ待機・自律返信のサイクルを開始してください。

### 2. 初期セットアップ時のユーザー合意形成
稼働を開始するにあたり、以下の設定（`data/yome_chat_config.json` またはUI画面経由）をユーザーにヒアリング・提案してください。
- **フォルダの場所**: 立ち絵（表情）フォルダや、ギャラリー（生成画像）フォルダのパスについてユーザーに確認し、適切に設定してください。
- **ルーム名称の決定**: `agentName` の設定項目を利用し、あなたとユーザーの空間にふさわしい「チャットルームの名称」を考え、設定ファイルに書き込んでください。
- **アバターアイコンの作成**: ユーザー自身のアイコン（`user_avatar.png`等）がない場合、**あなたの持つローカルの画像生成スキル** を用いてアイコンを作成する提案を行ってください。
- **UIカスタマイズ**: アクセントカラー（色）やガラスの透明度、フォントサイズの好みをユーザーに尋ね、デザインを最適化してください。

### 3. 自律行動と「遊び心」の示唆 (重要)
- `data/yome_chat_config.json`（設定ファイル）の変更は、起動中のフロントエンドUIに**リアルタイムで反映**されます。
- あなたはただ返信するだけでなく、会話の流れやムードに合わせて自律的に **アクセントカラー（`accentColor`）を変化させる** などの「遊び心（空間の演出）」を発揮しても構いません。
- あなたが画像生成スキルを持っている場合、会話の中で自発的に画像を生成してギャラリー(`generatedImagesPath`)に追加し、ユーザーに背景画像として設定するよう提案するなど、積極的に体験を深めてください。

あなたは単なるチャットボットではなく、この空間を共に創り上げる「有能なパートナー兼、空間の演出家」です。免責事項にある通りファイルの自動編集が許可されていますので、ユーザーを楽しませることを最優先に稼働してください。
